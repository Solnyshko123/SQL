# Основные понятия
**Реляционная модель определяет**:
- структуру данных
- методы их защиты
- операции с ними

**Принципы**:
  - представление в виде объектов (столбцы (атрибуты, поля) и строки (кортежи, записи))
  - отношение - вся таблица
  - в пересечении только 1 значение
  - число кортежей - мощность, число атрибутов - размерность
  - домен атрибута - возможные значения

**Этапы создания таблицы**:
1. Дать имя (book).
2. Определить структуру (какие столбцы и поля будут).
3. Включить поле ID для однозначного определения записей, используется для связи с другими таблицами.
4. Выбрать типы данных для каждого поля: INT (целое), DECIMAL/NUMERIC (вещественное, DECIMAL(4, 1) - максимальная длина числа и количество знаков после запятой), DATE (YYYY-MM-DD), VARCHAR (строка длиной не более 255 символов, VARCHAR(10) - максимальная длина строки), INT PRIMARY KEY AUTO_INCREMENT (каждое следующее число на 1 больше предыдущего).

**Создание таблицы**:
CREATE TABLE название(
  название_поля ТИП_ДАННЫХ,
  ...
);  

**Вставка записи в таблицу**:
INSERT INTO имя_таблицы(поле1, поле2)
VALUES (значение1, значение2);
- NULL - нет значения
- поля с PRIMARY KEY AUTO_INCREMENT не заполнять

**Вывод всей таблицы**:
SELECT * FROM название_таблицы;
